## GitHub Copilot Lab

### What Are Copilot Spaces?

Copilot Spaces organize task-specific code, documents, and instructions so Copilot can answer questions with the right context.

### Benefits of Copilot Spaces

Through Copilot Spaces you can:
- Let Copilot read the files, design docs, and custom instructions you attach, turning it into a domain expert for the task.
- Share spaces across an organization to replicate best practices and get new teammates up to speed quickly.
- Sync repositories automatically so Copilot always cites up-to-date knowledge.

### How We'll Use Them in This Lab

We will:
- Create and use a Copilot Space.
- Access the space from the IDE through MCP.

## Lab Steps

### Step 1: Create and Use Copilot Spaces

#### 1.1 Goal
Create a space and interact with it.

#### 1.2 Procedure

1. **Open the Copilot Spaces portal**  
   Visit `https://github.com/copilot/spaces`, choose **Spaces** in the sidebar, and click **Create space**.
   ![Copilot reference example](../images/121.png)

2. **Fill in the space details**  
   In the **New Space** dialog enter a name (for example, `testspace`) and click **Create Space**. The owner can be either an individual account or an organization.
   ![Copilot reference example](../images/122.png)

3. **Add repository files**  
   Inside the new space, click **Add sources → Add files from repository**.
   ![Copilot reference example](../images/123.png)

4. **Choose the repository**  
   Search for `jacwu/github-materials`, select it, and continue.
   ![Copilot reference example](../images/124.png)

5. **Select files to sync**  
   In the tree, select `copilot-ide-features.md` and `copilot-web-features.md` under the `github-copilot-features-status` folder, then click **Add**.
   ![Copilot reference example](../images/125.png)

6. **Ask Copilot inside the space**  
   In the space chat, ask “which ide support agent mode” and verify the response cites `copilot-ide-features.md`.
   ![Copilot reference example](../images/126.png)

#### 1.3 Validation
- The space lists the two synced documents with the status showing up-to-date.
- Copilot’s answer includes the IDE list (VS Code, Visual Studio, JetBrains, Xcode, Eclipse) and cites `copilot-ide-features.md`.

### Step 2: Access Copilot Spaces in the IDE

#### 2.1 Goal
Access Copilot Spaces in VS Code through MCP.

#### 2.2 Procedure

1. **Register the Copilot Spaces MCP service**  
   Add the following configuration to `mcp.json` and start the `copilot-spaces` MCP service:
   ```json
   "copilot-spaces": {
     "type": "http",
     "url": "https://api.githubcopilot.com/mcp/x/copilot_spaces"
   }
   ```
   ![Copilot MCP configuration](../images/127.png)

2. **List available spaces**  
   In VS Code Copilot Chat (Agent mode) run `#list_copilot_spaces` and wait for the `copilot-spaces` server to return the list.
   ![List Copilot Spaces](../images/128.png)

3. **Query a specific space**  
   Next run `query from copilot spaces: testspace, answer question: is coding agent available in copilotenterprise` and confirm the answer cites `copilot-web-features.md`.
   ![Query Copilot Space](../images/129.png)

#### 2.3 Validation
- `#list_copilot_spaces` returns the available spaces.
- Copilot responses reference the synced documents from the selected space.
